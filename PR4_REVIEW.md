## Issues to address before merge

### 1. Duplicate prerender route list (high)

- **`src/prerender-routes.ts`** exports `prerenderRoutes`.
- **`scripts/prerender.mjs`** has its own hardcoded `prerenderRoutes` array and does **not** import from the TS file (Node runs `.mjs` and can’t load `.ts` directly).
- **PRERENDER.md** says: “Add the route to `src/prerender-routes.ts`” – but the script only uses its own list.
- **Risk:** Adding a route only in `prerender-routes.ts` means it won’t be prerendered; the two lists can drift.

**Options:**

- **A (quick):** Add a short comment in both files: “Keep in sync with `scripts/prerender.mjs`” / “Keep in sync with `src/prerender-routes.ts`”, and in PRERENDER.md mention that **both** must be updated when adding a route.
- **B (better):** Run the prerender script with `tsx` and import `prerenderRoutes` from `../src/prerender-routes.ts` so there is a single source of truth (and update `package.json` `build:prerender` to use `tsx` for the script).

### 2. Report HTML: optional `image` / `link` (low)

- **`ContentMeta`** has `image?: string` and `link?: string`.
- **`scripts/generate-report-html.ts`** uses `report.image` and `report.link` without checks.
- If a report had no `image` or `link`, you’d get `"undefined"` in meta, `img src`, and `href`.

**Fix:** Add guards (e.g. skip or use a fallback) when `report.image` or `report.link` is missing. All current reports have both; this is for future-proofing.

### 3. Prerendered HTML `lang` always `sv` (low)

- In **`scripts/prerender.mjs`** the output template uses `<html lang="sv">` for every route.
- `/en/*` routes are therefore served with `lang="sv"`.

**Fix:** Set `lang` from the route, e.g. `lang="${route.startsWith('/en') ? 'en' : 'sv'}"` in the template.
